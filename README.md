# Move Car Page

一个纯静态的扫码挪车页面和海报生成器。

扫码后打开联系页，用户可点击按钮：

- 电话通知
- 短信通知

项目里的默认手机号已改为示例号 `13800138000`，发布前请替换成你自己的号码。

## 文件

- `index.html`：扫码后打开的联系页
- `generator.html`：海报生成器，输入手机号后生成高清二维码海报
- `.nojekyll`：让 GitHub Pages 直接按静态站点托管

## 默认逻辑

- 联系页默认地址示例：`https://car.dxboy.xyz`
- 二维码目标格式：`https://car.dxboy.xyz?phone=13800138000`
- 如果 URL 中带有 `phone` 参数，页面会优先使用该手机号
- 如果 URL 中没有 `phone` 参数，页面会使用示例号 `13800138000`

## 修改手机号

如果你只是临时自用，可以直接在生成器里输入手机号生成海报。

如果你想改项目默认示例号，请修改：

- `index.html` 中的默认回退号码
- `generator.html` 中输入框的默认值

## 本地测试

可以直接本地打开：

- `generator.html`
- `index.html`

建议用手机测试联系页按钮行为，桌面浏览器不适合验证拨号和短信能力。

## 部署

这是一个纯静态项目，可以部署到任意静态托管平台，例如：

- GitHub Pages
- EdgeOne Pages
- OSS 静态托管

## 说明

网页可以拉起拨号界面和短信编辑界面，但通常不能绕过系统确认直接自动拨出或自动发送。
